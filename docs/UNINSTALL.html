What the add-on writes to your hub

Home Assistant does not give add-ons an automatic cleanup hook on uninstall, so the files below must be removed manually if you want a full clean removal.

What Location Removed by UI uninstall?
Add-on container & image Docker (internal) ✓ Yes
Dark theme file /config/themes/cn_dark.yaml ✗ No
Light theme file /config/themes/cn_light.yaml ✗ No
About panel JS /config/www/cn-about-panel.js ✗ No
Branding scripts /config/www/cn-init.js, cn-scrubber.js, cn-hide-leaks.js ✗ No
What's New data /config/www/cn-whats-new.json ✗ No
card-mod library /config/www/card-mod.js ✗ No
configuration.yaml block /config/configuration.yaml (frontend + panel_custom) Manual edit
Lovelace resource HA Lovelace resources table Manual removal
Setup marker /config/.cn_frontend_setup_done ✗ No

Step 1 — Remove the Add-on

All users start here
  1. 1

    In Home Assistant, go to Settings → Add-ons

  2. 2

    Click Connect Nest FrontEnd in your installed add-ons list

  3. 3

    Scroll to the bottom and click Uninstall. Confirm when prompted.

    The add-on stops immediately and the container is removed. Your hub reverts to the standard Home Assistant interface.

ℹ️
Reinstalling or upgrading?

You're done at this step. Go back to the Add-on Store and install the new version. The setup marker file means run.sh won't re-patch configuration.yaml, but it will always refresh the theme and JS files.

Full Clean Removal

Steps 2 – 7
⚠️
You need file system access

Use the File Editor add-on, Studio Code Server, or SSH to complete these steps. If you don't have one of these installed, install the File Editor add-on from the Add-on Store first.

Step 2 — Remove theme files

Delete these two files from your hub's /config/themes/ folder:

  • 🎨/config/themes/cn_dark.yaml
  • 🎨/config/themes/cn_light.yaml
💡

In File Editor: navigate to config/themes/ → click the three-dot menu next to each file → Delete

Step 3 — Remove panel and branding scripts

Delete these files from /config/www/:

  • 📄/config/www/cn-about-panel.js
  • 📄/config/www/cn-init.js
  • 📄/config/www/cn-scrubber.js
  • 📄/config/www/cn-hide-leaks.js
  • 📄/config/www/cn-whats-new.json
  • 📄/config/www/card-mod.js

Step 4 — Edit configuration.yaml

Open /config/configuration.yaml and remove the Connect Nest block. It looks exactly like this:

Remove this entire block (including the comment lines)
# ─── Connect Nest Frontend ───────────────────────────────── # Auto-added by the Connect Nest Frontend add-on. # Edit with care — incorrect changes will break the branded UI. frontend: themes: !include_dir_merge_named themes extra_module_url: - /local/card-mod.js - /local/cn-init.js panel_custom: - name: cn-about sidebar_title: About Connect Nest sidebar_icon: mdi:information-outline url_path: cn-about module_url: /local/cn-about-panel.js embed_iframe: false require_admin: false # ─── End Connect Nest Frontend ─────────────────────────────
🚨
YAML is sensitive to indentation

Make sure no extra blank lines, spaces, or indentation errors remain after deleting the block. A malformed configuration.yaml will stop Home Assistant from starting.

After saving, use Developer Tools → Check Configuration to validate before restarting.

Step 5 — Delete the setup marker

This hidden file tells the add-on that first-run setup has already been done. Delete it so a future reinstall works cleanly.

  • 🗂️/config/.cn_frontend_setup_done
💡

In File Editor: click the gear icon → enable Show hidden files → navigate to config/ → delete .cn_frontend_setup_done

Via SSH: rm /config/.cn_frontend_setup_done

Step 6 — Remove the Lovelace resource

The add-on registered card-mod.js as a Lovelace resource via the HA API. Remove it from the dashboard resources list.

  1. 1

    Go to Settings → Dashboards

  2. 2

    Click the three-dot menu ⋮ (top right) → Resources

    💡

    If Resources is not visible, go to your Profile and enable Advanced Mode first.

  3. 3

    Find /local/card-mod.js in the list and click the bin 🗑 icon to delete it

Step 7 — Restart Home Assistant Core

  1. 1

    Go to Developer Tools (or Settings → System → Restart)

  2. 2

    Click Restart Home Assistant and wait approximately 60 seconds

Verify the removal is complete

After restart, tick each item to confirm everything is clean:

  • Sidebar no longer shows "About Connect Nest"
  • Theme dropdown (Profile page) no longer lists cn_dark or cn_light
  • Browser tab says "Home Assistant", not "Connect Nest"
  • Settings → Dashboards → Resources — no /local/card-mod.js or /local/cn-* entries
  • HA logs show no errors related to missing cn-init.js or cn-about-panel.js

Reinstalling after a full cleanup

✅

After completing all 7 steps, reinstalling the add-on works exactly like a brand new install. With the marker file gone, run.sh will re-patch configuration.yaml and re-copy all theme and panel files automatically.
